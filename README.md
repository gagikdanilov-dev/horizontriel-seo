# HorizonTriel — генератор статических SEO-страниц

Тянет опубликованные объекты из твоего PocketBase и собирает **реальные HTML-страницы**,
которые видят и Google, и Яндекс (в отличие от клиентского рендера на Тильде).

Что генерируется:

| Страница | URL | Под какой запрос |
|---|---|---|
| Хаб-каталог | `/` | «недвижимость по России» |
| Тип по РФ | `/kvartiry/`, `/doma/`, `/uchastki/`, `/garazhi/` … | «купить квартиру» |
| Город | `/gorod/kislovodsk/` | «недвижимость в Кисловодске» |
| Регион | `/region/stavropolskiy-kray/` | «недвижимость в Ставропольском крае» |
| **Город × тип** | `/gorod/kislovodsk/kvartiry/` | **«купить квартиру в Кисловодске»** ← самые конверсионные |
| Объект | `/obekt/{slug}/` | карточка с `RealEstateListing` + `Offer` |
| `sitemap.xml`, `robots.txt` | | для вебмастеров |

Кластеры создаются **только там, где есть хотя бы один объект** — никаких пустых
дублей-«дорвеев». Тексты и FAQ на каждой странице собираются из твоих же данных
(количество, диапазон цен, средняя площадь), поэтому уникальны.

---

## 0. Быстрый локальный прогон

```bash
python3 build.py --sample     # соберёт dist/ на тестовых данных
# открой dist/index.html в браузере
```

Боевой прогон против реального PocketBase:

```bash
python3 build.py              # берёт PB_URL/коллекцию из config.py или из env
```

Только стандартная библиотека Python 3.8+, ставить ничего не нужно.

---

## 1. Домен (главный вопрос)

Тильда занимает `horizontriel.com`, поэтому статику вешаем на **поддомен**:

```
catalog.horizontriel.com   →   GitHub Pages
```

1. В `config.py` (или переменной `SITE_BASE`) укажи `https://catalog.horizontriel.com`.
2. У регистратора/DNS добавь CNAME-запись:
   `catalog` → `<твой-логин>.github.io`.
3. Файл `CNAME` в `dist/` генерится автоматически из `SITE_BASE`.

> Поддомен vs подпапка: подпапка (`horizontriel.com/catalog/`) чуть сильнее консолидирует
> вес, но на Тильде без reverse-proxy (Cloudflare Worker) её не сделать. Поддомен обе
> поисковые системы индексируют нормально — для старта берём его.

---

## 2. Публикация через GitHub

1. Залей папку в новый репозиторий.
2. **Settings → Pages → Source: GitHub Actions**.
3. **Settings → Secrets and variables → Actions → Variables** — добавь (по желанию,
   иначе возьмутся дефолты из `config.py`):
   - `SITE_BASE` = `https://catalog.horizontriel.com`
   - `PB_URL` = `https://api.horizontriel.com`
   - `MAIN_SITE` = `https://horizontriel.com`
4. Вкладка **Actions → Build & Deploy → Run workflow** — первый запуск вручную.

Дальше воркфлоу сам пересобирает сайт **каждые 6 часов** (свежие объекты из PocketBase)
и при каждом пуше в `build.py`/`templates.py`/`config.py`. Частоту меняешь в `cron`.

> Если у тебя закрыт публичный доступ к коллекции `listings` — добавь секрет
> `PB_TOKEN` и Authorization-заголовок в `fetch_listings()` (сейчас чтение идёт без
> токена, как на самом сайте).

## 3. Связать с Тильдой (готовые файлы)

В комплекте два файла с **уже внесёнными правками** — просто вставь их в
соответствующие HTML-блоки Тильды и опубликуй:

- **`главная_1_НОВЫЙ.txt`** — ссылки в подвале (`?city=`, `?type=`, `?region=`,
  `?intent=buy`) заменены на реальные URL каталога (`catalog.horizontriel.com/...`).
  `?intent=sell` ведёт на `#contact` (продажа — заявка, не каталог).
- **`главная_2_НОВЫЙ.txt`** — в `htApplyListingSeo` выставлен `noindex, follow`,
  чтобы модалка объекта на Тильде (`?object=id`) не конкурировала со статической
  страницей объекта. Базовая страница и фильтры остаются в индексе.

> Ссылки в подвале **не дадут 404**: генератор всегда создаёт страницы приоритетных
> городов/типов/регионов. Где есть объекты — страница в индексе и в sitemap; где пусто —
> отдаёт 200 с формой заявки и уходит в `noindex` (вне sitemap, без риска thin-content).
> Появятся объекты в городе — при следующей сборке страница сама наполнится и войдёт в индекс.

Если поддомен другой (не `catalog.horizontriel.com`) — сделай в этих двух файлах
поиск-замену `catalog.horizontriel.com` на свой и синхронно поменяй `SITE_BASE`.

Опционально: на карточках каталога рядом с открытием модалки можно повесить реальный
`href` на `/obekt/{slug}/` — тогда у робота будет прямой путь обхода, а шаринг поведёт
на индексируемую страницу.

---

## 4. Отправить в поисковики

Добавь **поддомен** (это отдельный сайт для вебмастеров) и загрузи sitemap:

- **Яндекс.Вебмастер** → добавить сайт `catalog.horizontriel.com` → Индексирование →
  Файлы Sitemap → `https://catalog.horizontriel.com/sitemap.xml`.
  Затем «Переобход страниц» для главной хаба и топ-кластеров.
- **Google Search Console** → добавить ресурс (URL-prefix) `https://catalog.horizontriel.com/`
  → Sitemaps → тот же адрес.

## Файлы

- `config.py` — источник PB, домен, карты типов/городов, транслитерация, формат цены.
- `templates.py` — HTML-шаблоны и CSS (в стиле основного сайта).
- `build.py` — загрузка, нормализация записей (как `pbToListing`), генерация, sitemap.
- `sample_listings.json`, `sample_settings.json` — данные для `--sample`.
- `.github/workflows/build.yml` — сборка и деплой.
